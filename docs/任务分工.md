# 动作闯关游戏项目 - 五人任务分工

## 项目概述

**项目周期**: 建议8-10周
**开发模式**: 模块化并行开发 + 联合集成测试
**每周会议**: 周五下午进行进度汇报和问题讨论

---

## 人员分工

### 成员A - **嵌入式硬件开发工程师**

#### 主要职责
负责STM32F103VE开发板与摄像头硬件连接、驱动开发和图像采集

#### 具体任务
1. **硬件连接与调试** (第1-2周)
   - 连接OV7670/OV7725摄像头模块到STM32开发板
   - 配置SCCB/I2C接口进行摄像头寄存器配置
   - 配置DCMI/DMA进行图像数据采集
   - 测试各引脚信号波形

2. **摄像头驱动开发** (第3-4周)
   - 编写OV7670/OV7725初始化代码
   - 实现图像分辨率配置(QVGA/VGA)
   - 实现亮度、对比度、曝光等参数调整
   - 编写JPEG压缩配置（如支持硬件JPEG）

3. **图像采集与传输** (第5-6周)
   - 实现DMA图像采集到缓冲区
   - 编写串口图像分包发送函数
   - 实现图像格式转换(YUV422 to RGB565/JPEG)
   - 优化传输速度，达到1-2帧/秒

4. **与上位机协议对接** (第7-8周)
   - 实现与Python服务器的串口通信协议
   - 处理拍照命令、参数配置命令
   - 发送图像数据包和状态信息
   - 与成员B进行联合调试

#### 交付物
- 完整的STM32工程代码
- 硬件接线图和引脚定义文档
- 摄像头配置参数表
- 通信协议实现说明

#### 联合调试
- **第7-8周**: 与成员B进行硬件-软件联调
- **第9周**: 与成员D进行端到端图像传输测试

---

### 成员B - **上位机通信与硬件接口工程师**

#### 主要职责
负责STM32与电脑之间的串口通信、图像接收和转发给处理模块

#### 具体任务
1. **串口通信模块开发** (第1-3周)
   - 基于PySerial实现串口连接管理
   - 实现波特率自适应(115200/921600)
   - 实现数据帧解析(帧头、命令字、数据长度、校验和)
   - 编写串口错误重连机制

2. **图像数据接收** (第4-5周)
   - 实现图像数据包缓存和重组
   - 处理丢包检测和重传请求
   - 将接收到的JPEG数据保存为文件
   - 实现接收进度回调

3. **通信协议封装** (第6-7周)
   - 实现串口命令封装(开始拍照、停止、配置)
   - 实现状态查询和心跳机制
   - 编写自动设备检测功能
   - 实现多串口设备扫描

4. **与MediaPipe对接** (第8-9周)
   - 将接收到的图像传递给姿势检测模块
   - 实现图像格式转换(如有需要)
   - 优化图像传输延迟
   - 与成员C进行接口联调

#### 交付物
- 串口通信模块(serial_manager.py完善)
- 图像接收和缓存模块
- 设备自动检测工具
- 通信协议文档
- 单元测试代码

#### 联合调试
- **第5-6周**: 与成员A进行硬件串口通信测试
- **第8-9周**: 与成员C、D进行图像传输链路联调

---

### 成员C - **MediaPipe姿势识别与游戏逻辑工程师**

#### 主要职责
负责MediaPipe姿势检测、动作匹配算法和游戏核心逻辑

#### 具体任务
1. **MediaPipe环境搭建** (第1-2周)
   - 安装MediaPipe和相关依赖
   - 测试姿势检测示例代码
   - 了解姿势关键点输出格式
   - 编写MediaPipe初始化代码

2. **姿势检测模块开发** (第3-5周)
   - 实现图像输入接口(从成员B接收图像)
   - 编写姿势关键点检测函数
   - 实现关键点坐标归一化处理
   - 添加检测置信度过滤
   - 编写关键点可视化(用于调试)

3. **动作匹配算法** (第6-7周)
   - 设计动作模板存储格式
   - 实现关键点距离计算算法
   - 编写角度计算函数(关节角度)
   - 实现加权姿势相似度计算
   - 编写动作匹配结果判定

4. **游戏逻辑实现** (第8-9周)
   - 实现动作序列管理
   - 编写分数计算函数
   - 实现关卡进度追踪
   - 编写游戏状态机
   - 与成员D进行接口对接

#### 交付物
- MediaPipe姿势检测模块(pose_detector.py完善)
- 动作匹配算法模块
- 游戏逻辑核心(game_manager.py完善)
- 动作模板JSON文件
- 算法测试和调优报告

#### 联合调试
- **第5周**: 与成员B进行图像输入测试
- **第8-9周**: 与成员D进行姿势检测结果对接

   - 实现准备按钮和状态同步

#### 交付物

#### 联合调试

## 联合调试计划

### 阶段一: 模块内部测试 (第5-6周)
- 各成员完成各自模块开发
- 进行单元测试和接口验证
- 编写模块使用文档

### 阶段二: 两两联调 (第7-8周)

| 周次 | 联调组合 | 调试内容 |
|------|----------|----------|
| 第7周 | A ↔ B | 串口通信、图像传输 |
| 第7周 | B ↔ C | 图像输入、姿势检测 |
| 第7周 | C ↔ D | 检测结果传递、分数计算 |
| 第8周 | D ↔ (A+B+C) | 完整游戏流程 |

### 阶段三: 系统集成测试 (第9-10周)

#### 第9周: 系统联调
- **目标**: 完成端到端完整流程测试
- **测试内容**:
  1. 用户登录 → 关卡选择 → 游戏开始
  2. 倒计时 → 拍照 → 图像传输
  3. 姿势检测 → 动作匹配 → 结果显示
  4. 分数计算 → 下一动作/游戏结束
- **问题修复**: 记录并修复发现的问题

#### 第10周: 优化和验收
- **性能优化**: 减少延迟、提高帧率
- **用户体验优化**: 界面美化、动画流畅
- **稳定性测试**: 长时间运行测试
- **最终验收**: 五人一起进行完整测试

## 沟通协作机制

### 1. 定期会议
- **频率**: 每周五下午 2小时
- **形式**: 线下/线上结合
- **议程**:
  1. 各成员汇报本周进展(每人10分钟)
  2. 演示可运行的功能
  3. 讨论遇到的问题(集体解决)
  4. 制定下周计划
  5. 需要协调的事项

### 2. 日常沟通
- **即时通讯**: 微信群/QQ群(快速问题)
- **文档共享**: 腾讯文档/飞书(接口定义、会议纪要)
- **代码管理**: Git仓库(GitHub/Gitee)

### 3. 代码协作流程
```
1. 每个人在各自分支开发:  feature/成员名/功能名
2. 完成自测后提交PR到 develop 分支
3. 其他成员Code Review
4. 合并后删除特性分支
5. 联合调试时统一从 develop 分支构建
```

### 4. 接口定义管理
- **文档位置**: `/docs/api_protocol.md` (本文件)
- **修改流程**:
  1. 提出修改需求
  2. 相关成员讨论确认
  3. 统一更新文档
  4. 各成员按新接口实现

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| STM32摄像头驱动调试困难 | 高风险 | 成员A提前开始，准备备用方案(USB摄像头) |
| MediaPipe姿势检测精度不足 | 中风险 | 成员C预留调优时间，准备简化版匹配算法 |
| 串口传输速度不够 | 中风险 | 成员B尝试提高波特率，或压缩图像 |
| 前后端接口不匹配 | 中风险 | 第4周开始每周联调，及时调整接口 |
| 成员时间安排冲突 | 低风险 | 建立备用沟通机制，文档化所有决策 |
